## Reviewer Verdict (gpt-5.4 @ high) ‚Äî APPROVED_WITH_CHANGES

Plan is structurally sound. 10 required changes before build. No optional suggestions ‚Äî all are correctness blockers.

### Required Changes

| # | Location | Issue | Required fix |
|---|---|---|---|
| 1 | validator.py packet schema | Missing actor model identities in packet shape ‚Äî prior actor model IDs not carried forward | Packet must carry `drafter_model_id`, `reviewer_model_id`, and all prior actor identities. Validator checks all of them, not just next-tier identity |
| 2 | validator.py no-self-approval | Python chained `!=` (`a != b != c`) is not pairwise ‚Äî `a == c` passes | Replace with explicit pairwise: `assert a != b and b != c and a != c` for every triple |
| 3 | validator.py tier ordering | `valid_tier` check accepts any integer ‚Äî invalid tiers (0, 5, 99) silently pass | Constrain to `TIER_SEQUENCE = [1, 2, 3, 4]`; reject anything not in the list |
| 4 | validator.py planning_floor constants | Includes `gpt-5.4-codex-spark` (not a real model) | Remove; valid planning-floor models: `claude-opus-4-6`, `claude-sonnet-4-6`, `gpt-5.4`, `gpt-5.3-codex-spark` |
| 5 | validator.py planning_floor check | Planning-floor check applies to every tier actor | Bind explicitly to Tier 1 actor only; Tier 2+ workers are not subject to the planning floor |
| 6 | validator.py PII floor | PII floor validated at packet entry only | Must be re-validated at every model boundary in the packet chain ‚Äî if any intermediate model is not LAM-family, the invariant fails |
| 7 | validator.py missing fields | Missing required fields use silent defaults (`get(key, default)`) | Missing required fields must raise `ValidationError`; no silent defaults on schema-required keys |
| 8 | validator.py fallback log | Fallback path validity checked by `len(path) > 0` only | Must check that the path resolves to a valid directory under `raw/model-fallbacks/` (parent dir exists, not just non-empty string) |
| 9 | server.py som.delegate | `som.delegate` reads parent chain then writes ‚Äî window between read and write | Use an atomic write: acquire a file-level lock on the chain file before read+write, release after. `fcntl.flock` or `filelock` acceptable |
| 10 | tests | Test suite covers positive path only for most invariants | Add explicit negative test for each of the 9 invariant changes above; all must raise `ValidationError` or equivalent |
