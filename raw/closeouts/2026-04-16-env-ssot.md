## Decision Made
Established `hldpro-governance/.env.shared` as the single source of truth for all operator credentials across all HLDPRO repos, with a bootstrap script to generate repo-native `.env.local` files from it.

## Pattern Identified
Credential sprawl across repos (`AIS .env`, `LAM .env`, `HP frontend/.env.staging`) caused worktree env failures and made rotation painful. The SSOT + bootstrap pattern (one vault, generated artifacts) prevents this class of "missing env on clean checkout / new worktree" errors.

## Contradicts Existing
- `docs/EXTERNAL_SERVICES_RUNBOOK.md` scope line previously said "Repo-local services live in each downstream repo's own runbook" — superseded. Governance is now SSOT for external services documentation too.
- `ai-integration-services/docs/EXTERNAL_SERVICES_RUNBOOK.md` and `HealthcarePlatform/docs/EXTERNAL_SERVICES_RUNBOOK.md` — still exist but should be treated as stubs; governance runbook §9–25 is now authoritative.
