# Fail-Fast Log

> Governance-repo error patterns and resolutions.
> Read this before retrying a failing overlord workflow, helper command, or Codex ingestion run.

## Format

Use this table format:

| Error | Root Cause | Resolution | Date |
|-------|-----------|------------|------|

## Category: TOOLING — Codex CLI, scripts, and operator helpers

Keep the CLI sandboxed; remove the dependency on in-sandbox `git diff` discovery. | 2026-04-06 |
| Precomputed diff context fixed sandbox discovery, but large repos started failing with `Argument list too long` | The helper initially embedded the bounded patch excerpt directly into the positional `codex exec` prompt argument. | graphify git-hook install fails from a git worktree | `graphify hook install` assumes `.git/hooks` is a real directory. | Knowledge-base automation called an interactive-only graphify entrypoint | The source plan and merged automation used `graphify . --update --no-viz`, but the installed CLI exposes install/query/hook commands while the actual build flow lives behind the interactive `/graphify` skill. | Governance backlog drifted into a shadow local execution queue | `OVERLORD_BACKLOG.md` held active work items without GitHub issue references, so the repo had no hard distinction between roadmap/status notes and the actual execution backlog.
